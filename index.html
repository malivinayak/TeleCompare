About TeleCompare

How the data works

Plan data is stored as a static data/plans.json file, updated automatically by a GitHub Actions workflow that runs daily. The frontend reads this file directly — no backend server required.

Data update pipeline

  • A scheduled GitHub Actions workflow runs scripts/update_plans.py daily
  • The script fetches from available public sources and normalises the data
  • If a live fetch fails, the previous valid plans.json is preserved (fallback mode)
  • scripts/validate_plans.py validates the JSON before it is committed
  • Changes are committed back to the repository automatically, triggering a Pages redeploy

Important limitations

  • Plan availability and pricing vary by telecom circle (state)
  • Operator websites may change structure, which can affect automated scraping
  • Always verify the final price and benefits on the operator's official website before recharging
  • OTT benefits are subject to change by operators at any time
